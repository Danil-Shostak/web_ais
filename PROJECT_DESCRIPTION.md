# Описание проекта АИИО РБ

## Созданы основные модели данных системы (таблицы в Supabase PostgreSQL):

- **Модель "Logs"** для хранения информации о действиях пользователей в системе;
- **Модель "Notifications"** для хранения уведомлений пользователей;
- **Модель "Institutions"** для ведения реестра образовательных учреждений Республики Беларусь;
- **Модель "Reports"** для формирования и сохранения аналитических отчетов;
- **Модель "Staff"** для хранения списка работников образовательных учреждений;
- **Модель "Students"** для хранения списка учащихся;
- **Модель "Profiles"** для хранения профилей пользователей с разграничением ролей;
- **Модель "Statistics"** для хранения статистических данных по учреждениям.

## Реализована система отчетности:

- **Модуль "reports.js"** для бизнес-логики генерации отчетов в форматах PDF, Excel и Word;
- **Сервис "api.js"** для работы с облачной базой данных Supabase (CRUD операции);
- **Модуль "utils.js"** с вспомогательными функциями для форматирования и обработки данных;
- **Модуль "validators.js"** для валидации пользовательского ввода.

## Разработан современный пользовательский интерфейс:

- **Страница "reports.js"** для работы с отчетами и их генерации;
- **Страница "institutions.js"** для управления каталогом учреждений образования;
- **Страница "dashboard.js"** для отображения общей статистики и ключевых показателей;
- **Страница "students.js"** для управления списком учащихся;
- **Страница "staff.js"** для управления персоналом учреждений;
- **Страница "statistics.js"** для просмотра статистических графиков и диаграмм;
- **Страница "import.js"** для массовой загрузки данных из Excel/CSV файлов;
- **Страница "settings.js"** для управления настройками профиля;
- **Страница "admin.js"** для администрирования системы.

## Архитектура приложения

Архитектура приложения построена как **SPA (Single Page Application)** на чистом JavaScript без использования фреймворков. Для хранения данных используется облачная база данных **Supabase (PostgreSQL)** с механизмом **Row Level Security (RLS)** для разграничения доступа. Приложение реализует модульную структуру с разделением на страницы и сервисы, что обеспечивает удобство сопровождения и масштабирования.

## Основные компоненты системы

| Компонент | Описание |
|-----------|----------|
| `api.js` | API-слой для взаимодействия с Supabase |
| `auth.js` | Система аутентификации и авторизации |
| `config.js` | Конфигурация приложения |
| `supabase-client.js` | Клиент Supabase |
| `utils.js` | Утилиты и вспомогательные функции |
| `validators.js` | Валидаторы данных |
| `app.js` | Основной файл приложения |

## Технологический стек

- **Frontend**: HTML5, CSS3, JavaScript (ES6+)
- **База данных**: Supabase (PostgreSQL)
- **Библиотеки**:
  - Chart.js — построение графиков и диаграмм
  - jsPDF — генерация PDF-документов
  - SheetJS (XLSX) — работа с Excel-файлами
  - FileSaver.js — скачивание файлов
- **Хостинг**: GitHub Pages
