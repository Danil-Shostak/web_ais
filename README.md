# АИИО РБ - Автоматизация аналитической информации учреждений образования

Веб-приложение для дипломного проекта по автоматизации аналитической информации учреждений образования Республики Беларусь.

## Описание

Это полноценное SPA-приложение, разработанное для управления и аналитики данных учреждений образования Беларуси. Приложение включает:

- ✅ Систему регистрации и авторизации пользователей
- ✅ Разграничение ролей (администратор, редактор, пользователь)
- ✅ Каталог учреждений образования с полным CRUD
- ✅ Управление списками учащихся и работников
- ✅ Статистические графики и диаграммы
- ✅ Генерация отчетов в PDF, Excel, Word
- ✅ Импорт данных из Excel/CSV файлов
- ✅ Логирование действий пользователей
- ✅ Адаптивный дизайн для мобильных устройств

## Технологии

- **Frontend**: HTML, CSS, JavaScript (без фреймворков)
- **База данных**: Supabase (PostgreSQL)
- **Хостинг**: GitHub Pages
- **Библиотеки**:
  - Chart.js - графики
  - jsPDF - PDF отчеты
  - SheetJS (XLSX) - Excel файлы
  - FileSaver.js - скачивание файлов

## Установка и настройка

### 1. Настройка Supabase

1. Зарегистрируйтесь на [supabase.com](https://supabase.com)
2. Создайте новый проект
3. Перейдите в SQL Editor и выполните код из файла `supabase/schema.sql`
4. Скопируйте URL проекта и anon ключ

### 2. Настройка приложения

Откройте файл `js/config.js` и замените значения на свои:

```javascript
supabase: {
    url: 'https://your-project.supabase.co',
    anonKey: 'your-anon-key-here'
}
```

### 3. Деплой на GitHub Pages

1. Создайте репозиторий на GitHub
2. Загрузите все файлы проекта
3. Перейдите в Settings → Pages
4. Выберите ветку (main) и папку (/)
5. Нажмите Save

## Использование

### Регистрация и вход

1. Откройте приложение
2. Нажмите "Регистрация"
3. Введите email и пароль
4. После регистрации вы получите роль "Пользователь"

### Роли пользователей

- **Администратор**: полный доступ ко всем функциям
- **Редактор**: может добавлять и редактировать данные
- **Пользователь**: только просмотр

### Основные функции

1. **Дашборд** - обзор системы и основные показатели
2. **Учреждения** - каталог учреждений образования
3. **Учащиеся** - управление списком учащихся
4. **Работники** - управление персоналом
5. **Статистика** - графики и аналитика
6. **Отчеты** - генерация отчетов
7. **Импорт** - массовая загрузка данных
8. **Настройки** - профиль и безопасность

## База данных

Создано **10 связанных таблиц**:

1. `profiles` - профили пользователей
2. `institutions` - учреждения образования
3. `students` - учащиеся
4. `staff` - работники
5. `statistics` - статистические данные
6. `reports` - отчеты
7. `logs` - логи действий
8. `notifications` - уведомления
9. `settings` - настройки
10. `institution_contacts` - контакты учреждений
